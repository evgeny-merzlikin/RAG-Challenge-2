# Объяснение RAG Challenge Winner Solution

Этот репозиторий содержит победившее решение конкурса RAG Challenge - системы для ответов на вопросы о годовых отчетах компаний. Система использует передовые техники RAG (Retrieval-Augmented Generation) для точного извлечения и анализа информации из PDF документов.

## Общая архитектура системы

Система состоит из нескольких ключевых компонентов, работающих в pipeline:

1. **Парсинг PDF** → 2. **Обработка текста** → 3. **Векторизация** → 4. **Поиск** → 5. **Генерация ответов**

## Основные модули

### 1. Pipeline (`src/pipeline.py`)

Центральный модуль, координирующий весь процесс обработки:

#### Конфигурации системы
- **PipelineConfig**: Управляет путями к файлам и директориям
- **RunConfig**: Настройки алгоритмов (векторный поиск, reranking, модели LLM)

#### Основные этапы pipeline:
```python
# Парсинг PDF файлов
pipeline.parse_pdf_reports()

# Обработка таблиц (опционально)
pipeline.serialize_tables()

# Объединение и упрощение структуры
pipeline.merge_reports()

# Разбиение на чанки
pipeline.chunk_reports()

# Создание векторных баз данных
pipeline.create_vector_dbs()

# Обработка вопросов
pipeline.process_questions()
```

#### Предустановленные конфигурации:
- `max_nst_o3m_config` - Лучшая конфигурация с OpenAI o3-mini
- `ibm_llama70b_config` - Конфигурация с IBM Llama 70B
- `gemini_thinking_config` - Конфигурация с полным контекстом Gemini

### 2. Парсинг PDF (`src/pdf_parsing.py`)

Модуль для извлечения структурированной информации из PDF:

#### Ключевые особенности:
- Использует библиотеку **Docling** для парсинга
- Поддерживает параллельную обработку нескольких PDF
- Извлекает текст, таблицы, изображения с метаданными
- Нормализует последовательность страниц

#### Основные классы:
- **PDFParser**: Основной класс для парсинга PDF
- **JsonReportProcessor**: Обработка и структурирование результатов

```python
# Пример использования
parser = PDFParser(output_dir=Path("./parsed_pdfs"))
parser.parse_and_export(doc_dir=Path("./pdf_reports"))
```

### 3. Обработка вопросов (`src/questions_processing.py`)

Модуль для обработки вопросов и генерации ответов:

#### Функциональность:
- Извлечение названий компаний из вопросов
- Поиск релевантного контекста в документах
- Обработка сравнительных вопросов (несколько компаний)
- Валидация ссылок на страницы
- Параллельная обработка вопросов

#### Основной класс QuestionsProcessor:
```python
processor = QuestionsProcessor(
    vector_db_dir="./vector_dbs",
    documents_dir="./documents", 
    parent_document_retrieval=True,  # Возврат полных страниц
    llm_reranking=True,              # LLM reranking
    top_n_retrieval=10               # Количество результатов
)
```

### 4. Поиск и извлечение (`src/retrieval.py`)

Система поиска релевантной информации:

#### Типы retriever'ов:
1. **VectorRetriever** - Векторный поиск с embeddings
2. **BM25Retriever** - Лексический поиск (TF-IDF)
3. **HybridRetriever** - Комбинированный подход с LLM reranking

#### Ключевые возможности:
- Поиск по названию компании
- Parent Document Retrieval (возврат полных страниц вместо чанков)
- Косинусное сходство для векторного поиска
- Интеграция с OpenAI embeddings

### 5. API запросы (`src/api_requests.py`)

Модуль для работы с различными языковыми моделями:

#### Поддерживаемые провайдеры:
- **OpenAI** (GPT-4o, o3-mini)
- **IBM Watson** (Llama модели)
- **Google Gemini** (Flash, Thinking)

#### Основные возможности:
- Структурированные ответы с Pydantic схемами
- Обработка ошибок и повторные попытки
- Подсчет токенов
- Асинхронная обработка запросов

### 6. Промпты (`src/prompts.py`)

Система промптов для различных типов вопросов:

#### Типы схем ответов:
- **Name** - Имена людей, компаний, продуктов
- **Number** - Численные значения с валидацией
- **Boolean** - Да/нет вопросы
- **Names** - Списки имен
- **Comparative** - Сравнительные вопросы

#### Особенности промптов:
- Chain-of-Thought reasoning (пошаговый анализ)
- Строгая валидация метрик
- Обработка валют и единиц измерения
- Защита от галлюцинаций

### 7. Дополнительные модули

#### Reranking (`src/reranking.py`)
- LLM-based переранжирование результатов поиска
- Оценка релевантности от 0 до 1
- Комбинирование векторных и LLM оценок

#### Text Splitter (`src/text_splitter.py`)
- Разбиение документов на чанки
- Сохранение контекста между чанками
- Поддержка различных размеров чанков

#### Tables Serialization (`src/tables_serialization.py`)
- Преобразование таблиц в текстовый формат
- Параллельная обработка таблиц
- Сохранение структуры данных

## Рабочий процесс системы

### 1. Подготовка данных
```bash
# Парсинг PDF файлов
python main.py parse-pdfs --parallel

# Обработка таблиц (опционально)
python main.py serialize-tables

# Создание баз данных
python main.py process-reports --config no_ser_tab
```

### 2. Обработка вопросов
```bash
# Запуск с лучшей конфигурацией
python main.py process-questions --config max_nst_o3m
```

### 3. Результаты
Система генерирует:
- **answers.json** - Финальные ответы для submission
- **answers_debug.json** - Детальная отладочная информация
- Статистику обработки (успех/ошибки/N/A ответы)

## Ключевые инновации

### 1. Custom PDF Parsing
- Использование Docling для высококачественного извлечения
- Нормализация структуры документов
- Обработка таблиц и изображений

### 2. Parent Document Retrieval
- Возврат полных страниц вместо мелких чанков
- Сохранение контекста для лучшего понимания

### 3. LLM Reranking
- Переоценка результатов поиска с помощью LLM
- Комбинирование семантического и лексического поиска

### 4. Query Routing
- Автоматическое определение типа вопроса
- Различные стратегии для разных схем ответов

### 5. Structured Output
- Использование Pydantic схем для валидации
- Chain-of-Thought reasoning в ответах
- Строгая валидация численных значений

## Конфигурации и настройка

### Основные параметры RunConfig:
- `use_serialized_tables` - Использование сериализованных таблиц
- `parent_document_retrieval` - Возврат полных страниц
- `llm_reranking` - LLM переранжирование
- `top_n_retrieval` - Количество результатов поиска
- `answering_model` - Модель для генерации ответов
- `api_provider` - Провайдер API (openai/ibm/gemini)

### Рекомендуемые конфигурации:
- **Лучшая точность**: `max_nst_o3m_config`
- **Большой контекст**: `gemini_thinking_config`
- **Альтернативная модель**: `ibm_llama70b_config`
